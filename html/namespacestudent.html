Plan the motion of pursuer and evader inside the arena.


Robot number 0 is given the role of evader.
Robot number 1 is given the role of pursuer.

The algorithm executes the following steps:

  1. Obstacles and borders are inflated to account for the robots encumbrance. The robots can then be handled as oriented points. See rm::inflate()
  2. A set of vertices are selected for the visibility graph, accounting for a distance from the inflated obstacles that allows for a higher chance of feasible Dubins curves existing. See rm::makeVisibilityNodes()
  3. A visibility graph is created as the base graph for the RoadMap. See rm::visibility()
  4. The RoadMap is built, pre-computing all feasible Dubins paths connecting nodes according to the base graph. See rm::RoadMap::build() and dubins
  5. The initial poses of the robots are added to the graph. See rm::RoadMap::addStartPose()
  6. The goal pose of each gate is computed and added to the graph. See rm::getGatePose() and rm::RoadMap::addGoalPose()
  7. Navigation maps to the goal poses are pre-computed. See nav::NavMap()
  8. The acutal pursuer-evader game takes place. See nav::runGame()
  9. The computed paths are discretized. See nav::discretizePath()
  10. The discretized paths are truncated at the point where the two robots collide, terminating the game. See nav::truncatePaths()
  11. [Optional] A matlab file containing functions to plot the arena configuration and the computed paths is generated. See utils::MatlabPlot
